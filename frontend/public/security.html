Security

Last updated: 2026-05-14 · Source repo: github.com/afstudy20-gif/wiz3

1. Architecture

  • All statistical computation runs on the backend (FastAPI + Python on Render).
  • The frontend is a static React build — it does NOT process the dataset.
  • TLS is terminated by Render at the edge. HSTS is set to one year with subdomain include.

2. Data handling

  • Uploaded files are parsed in RAM. They are never written to disk.
  • Backend session TTL is 30 minutes of inactivity (SESSION_TTL_SECONDS = 1800 in backend/services/store.py). Expired sessions are evicted by an in-process cleaner.
  • Maximum concurrent sessions is capped (MAX_SESSIONS = 20); oldest are evicted on overflow.
  • No long-term storage. No backup. No replication.

3. Browser-hardening headers

Every response carries:

4. Code-execution sandbox

Off by default. When enabled (ENABLE_CODE_RUNNER=1), submitted Python runs in an isolated subprocess with:

  • RLIMIT_CPU = run-timeout (max 60 s)
  • RLIMIT_AS = 512 MB
  • RLIMIT_FSIZE = 10 MB
  • RLIMIT_NOFILE = 64
  • RLIMIT_NPROC = 32
  • sys.meta_path import allowlist (numpy / pandas / scipy / statsmodels / lifelines / sklearn / matplotlib / seaborn + safe stdlib) — explicit deny-list for socket, http, subprocess, ctypes, os, shutil, pickle, ...
  • Linux: unshare --user --net when available — strips the network namespace
  • Matplotlib forced to Agg backend
  • Rate limits: per-session, per-IP, global, concurrency semaphore — all env-configurable (see SECURITY.md)
  • Audit log (backend/logs/code_runner.jsonl): timestamp, IP, session, SHA-256 of code, first 200 chars, duration, exit code — dataset never logged

5. Known limitations

  • The sandbox shares the host kernel — no Docker-level isolation (yet).
  • No defence against kernel-level exploits or speculative-execution side channels.
  • uSTAT has not yet been independently audited by a third party.

6. Automated security checks

Every push to main runs the following in CI (GitHub Actions .github/workflows/security-scan.yml):

  • bandit -r backend/ — Python static security analysis
  • pip-audit — Python dependency CVE check
  • npm audit --audit-level=high — npm dependency CVE check (frontend)
  • semgrep --config p/python --config p/typescript --config p/owasp-top-ten

7. Vulnerability disclosure

If you discover a security issue, please email adycovs@gmail.com with the prefix [uSTAT-security]. We commit to acknowledging within 5 business days. A machine-readable contact lives at /.well-known/security.txt (RFC 9116).

Please do not test for vulnerabilities against the production instance with anything other than read-only requests. Use a local fork for invasive testing.
