Security

Last updated: 2026-05-14 · Source repo: github.com/afstudy20-gif/wiz3

1. Architecture

  • All statistical computation runs on the backend (FastAPI + Python on Render).
  • The frontend is a static React build — it does NOT process the dataset.
  • TLS is terminated by Render at the edge. HSTS is set to one year with subdomain include.

2. Data handling

  • Uploaded files are parsed in RAM. They are never written to disk.
  • Backend session TTL is 30 minutes of inactivity (SESSION_TTL_SECONDS = 1800 in backend/services/store.py). Expired sessions are evicted by an in-process cleaner.
  • Maximum concurrent sessions is capped (MAX_SESSIONS = 20); oldest are evicted on overflow.
  • No long-term storage. No backup. No replication.

3. Browser-hardening headers

Every response carries:

HeaderValue
Strict-Transport-Securitymax-age=31536000; includeSubDomains; preload
Content-Security-Policyself + Plotly + inline tailwind; report-only until tuned
X-Frame-OptionsDENY — anti-clickjacking
X-Content-Type-Optionsnosniff
Referrer-Policystrict-origin-when-cross-origin
Permissions-Policycamera / mic / geo / payment / usb / sensors denied
Cross-Origin-Opener-Policysame-origin

4. Code-execution sandbox

Off by default. When enabled (ENABLE_CODE_RUNNER=1), submitted Python runs in an isolated subprocess with:

  • RLIMIT_CPU = run-timeout (max 60 s)
  • RLIMIT_AS = 512 MB
  • RLIMIT_FSIZE = 10 MB
  • RLIMIT_NOFILE = 64
  • RLIMIT_NPROC = 32
  • sys.meta_path import allowlist (numpy / pandas / scipy / statsmodels / lifelines / sklearn / matplotlib / seaborn + safe stdlib) — explicit deny-list for socket, http, subprocess, ctypes, os, shutil, pickle, ...
  • Linux: unshare --user --net when available — strips the network namespace
  • Matplotlib forced to Agg backend
  • Rate limits: per-session, per-IP, global, concurrency semaphore — all env-configurable (see SECURITY.md)
  • Audit log (backend/logs/code_runner.jsonl): timestamp, IP, session, SHA-256 of code, first 200 chars, duration, exit code — dataset never logged

5. Known limitations

  • The sandbox shares the host kernel — no Docker-level isolation (yet).
  • No defence against kernel-level exploits or speculative-execution side channels.
  • uSTAT has not yet been independently audited by a third party.

6. Verify our claims yourself

You don't have to trust us. Every claim on this page is verifiable from a terminal or a third-party scanner. Below are the exact commands and links.

Security headers

curl -I https://ustat.drtr.uk/api/health

You should see strict-transport-security (1-year preload), x-frame-options: DENY, x-content-type-options: nosniff, content-security-policy-report-only, referrer-policy: strict-origin-when-cross-origin, permissions-policy with the powerful APIs denied, and cross-origin-opener-policy: same-origin. The server: header is stripped.

Third-party scanners (one click, public)

  • Mozilla Observatory — full report
  • SecurityHeaders.com grade
  • Qualys SSL Labs — TLS configuration
  • HSTS Preload — status
  • VirusTotal — domain reputation

"Files never touch disk" — verify on a self-hosted run

Clone the repo, start the backend with strace or fs.watcher attached, and upload a file. No open(..., O_WRONLY) call hits a path under /app. Quick demo on Linux:

git clone https://github.com/afstudy20-gif/wiz3 && cd wiz3/backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
strace -e trace=openat -f -- uvicorn main:app --port 8000 \
    2>&1 | grep -E 'O_WRONLY|O_CREAT' | grep -v site-packages

The only writes you should see are framework / static-file caching outside the uploads/ path — there is no uploads/ path. services/store.py keeps everything in process RAM and evicts after SESSION_TTL_SECONDS = 1800 of inactivity.

Source-side verification

The relevant lines:

  • services/store.py — the in-RAM dict, the TTL cleaner, and the eviction policy.
  • routers/upload.py — the upload handler. No open(..., "w"), no to_csv(), no shutil.copy.
  • middleware/security_headers.py — every header value above is set by this file.
  • .github/workflows/security-scan.yml — the CI scan workflow.

7. Automated security checks

Every push to main runs the following in CI (GitHub Actions .github/workflows/security-scan.yml):

  • bandit -r backend/ — Python static security analysis
  • pip-audit — Python dependency CVE check
  • npm audit --audit-level=high — npm dependency CVE check (frontend)
  • semgrep --config p/python --config p/typescript --config p/owasp-top-ten

7. Vulnerability disclosure

If you discover a security issue, please email security@drtr.uk with the prefix [uSTAT-security]. We commit to acknowledging within 5 business days. A machine-readable contact lives at /.well-known/security.txt (RFC 9116).

Please do not test for vulnerabilities against the production instance with anything other than read-only requests. Use a local fork for invasive testing.
